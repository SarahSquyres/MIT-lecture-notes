Lecture 1: Introduction to Algorithms


What's an algorithm

  • Procedure that takes an input and produces a correct/valid output

Induction

  • Concept that involves using a base case and a recursive formula to solve a problem
  • Used in: functional programming, imperative programming, and logic programming
  • In a mathematical context: proving a statement is true for all positive integer
  • In programming:
    • Base case
    • defining a base case that solves the problem for a specific input or set of inputs
    • Recursive formula
    • define a recursive formula that solves the problem for a larger input or set of inputs, based on the solution to the base case
  • Steps of Induction in Programming
    1. Define the problem
      • identify a problem that can be solved using induction
      • calculating the factorial of a number
      • calculating the Fibonacci sequence
      • solving recursive data structures: trees and graphs
    2. Define the base case
      • define a base case that solves the problem
      • the factorial of 0 or 1
    3. Define the recursive formula
      • define a recursive formula that solves the problem for a larger input
      • the factorial of n = n (times the factorial of n-1)
    4. Apply the recursive formula
      • Apply the recursive formula to solve the problem for larger inputs or sets of inputs
      • see factorial.js

Birthday Problem

  1. maintain a record (array storage or store in object)
  2. interview each student to get their birthday data (recursive/iterate through elements)
    1. Check if birthday is in record
      • if is, return pair
    2. Add new student to record
  3. Else, return none

    Proof of Correctness

  • inductive hypothesis
    • if first "k" students contain match, return before interviewing student "k + 1"
  • base case
    • k = 0
    • assume inductive hypothesis true for k = k` (k prime)
      • if k` contains match, already returned by induction
      • else if k` + 1 contains match
      • algo checks k` + 1 against all students

    Proof of Efficiency

  • how fast does it run compared to other possible solutions
  • dont measure time, instead count operations
  • expect performance to depend on size of input
  • Big O notation - upper bounds
  • Ω (omega) - lower bounds
  • Θ (theta) - upper and lower
  • linear time algorithm
    • constant time: Θ(1)
      • performance of algorithm doesn't depend on input size
    • logarithmic time: Θ(log n)
      • runtime increases very slowly as n grows (efficient)
    • linear time: Θ(n)
      • execution time increases porportionally to size of input data
    • loglinear time: Θ(n log n)
      • algorithm's time complexity grows proportionally to the input size (n) multiplied by the logarithm of that size (log n)
      • as the input size increases, the operation count increases at a slower rate than a liner relationship but faster than a logarithmic one
    • quadratic time: Θ(n^2)
      • runtime increases proportionally to the square of the input size
    • polynomial time: Θ(n^c)
      • runtime increases at a rate proportional to the number of inputs
    • exponential time: 2^Θ(n)
      • runtime increases significantly with each additional unit added to the input

Model of Computation

  • Word-RAM
    • Random Access Memory: access different places in memory in constant time
    • think of RAM like a bunch of labeled boxes, each box can hold a small piece of data like a word or number
    • when the computer needs to use the data, it searches for the appropriate box
    • copies the data from the hard drive into the RAM so the CPU can use the data quickly and efficiently
    • address for each byte (8 bits) in memory
